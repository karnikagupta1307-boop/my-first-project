1. INTRODUCTION
================================================================================

Smart Parcel Management System is a comprehensive web-based application developed to streamline and automate the parcel handling process at educational institutions, particularly for student accommodations and campus mailrooms. In today's fast-paced educational environment, students frequently receive parcels from e-commerce platforms, families, and courier services. Managing these parcels manually creates significant administrative burden, increases risk of lost packages, and causes delays in parcel delivery to students.

Many educational institutions still rely on traditional paper-based or spreadsheet methods to track incoming parcels. This manual approach leads to several challenges including misplaced packages, lack of real-time tracking, difficulty in notifying students promptly, and inefficient verification during handover. Students often face inconvenience when collecting parcels, experiencing long wait times, uncertainty about parcel arrival, and concerns about package security.

The Smart Parcel Management System is designed to solve these problems by providing an intelligent, automated platform that digitizes the entire parcel lifecycle from arrival to delivery. The system maintains accurate records of all incoming parcels, automatically notifies students via email with OTP-based verification, and provides real-time tracking of parcel status. This ensures secure, efficient, and transparent parcel management for both administrators and students.

The application includes major features such as secure role-based authentication (Admin, Watchman/Staff, and Student), comprehensive parcel registration and tracking, automated OTP generation and email notification, real-time dashboard with statistics, student self-service portal, and detailed search and filtering capabilities. These features allow administrators to efficiently manage high volumes of parcels while providing students with instant visibility into their package status.

The frontend of the system is developed using HTML5, CSS3, and JavaScript (ES6+) to create a responsive, interactive, and user-friendly interface. The backend is developed using Node.js with Express.js framework for server-side processing, RESTful API development, and business logic implementation. MySQL is used for efficient relational database management and secure storage of student records, parcel data, and OTP verification logs. Nodemailer with Gmail SMTP is integrated for automated email notifications.

The project follows a Three-Tier Architecture consisting of the Presentation Layer, Application Logic Layer, and Database Layer. This architecture improves system maintainability, scalability, security, and performance while ensuring proper separation of responsibilities between frontend, backend, and database operations.

Smart Parcel Management System combines the concepts of Web Programming and Database Management Systems to create a practical real-world full-stack application. It serves as a useful platform for educational institutions, hostels, and residential complexes to modernize their parcel handling operations and improve service quality for residents.


1.1 Problem Statement

In the current educational institution environment, parcel management remains a largely manual and inefficient process. When parcels arrive at campus mailrooms or security gates, staff must manually record details in registers or spreadsheets, which is time-consuming and error-prone. Students are often unaware of their parcel arrivals until physical notices are posted or phone calls are made, causing delays in collection.

Traditional parcel handling methods present several critical issues:

- Manual record-keeping leads to data entry errors and lost package information
- Lack of real-time tracking makes it impossible to know parcel status instantly
- No automated notification system means students remain unaware of arrivals
- Verification during handover is weak, creating security risks
- Searching through historical records is difficult and time-consuming
- No centralized dashboard for administrators to monitor operations
- Students cannot self-serve to check their parcel status
- Paper-based systems are environmentally unfriendly and difficult to backup

Students face significant inconvenience due to these inefficiencies. They must physically visit the mailroom to check for parcels, wait in queues during peak times, and carry identification documents for verification. The absence of a digital system also prevents institutions from generating analytics and insights about parcel volumes, peak periods, and operational efficiency.

There is a strong need for a smart, automated system that can digitize parcel registration, provide real-time tracking, automatically notify recipients, ensure secure handover verification, and offer comprehensive administrative controls.

Smart Parcel Management System addresses this problem by offering a web-based intelligent parcel management platform that streamlines operations, enhances security through OTP verification, and provides a superior experience for both administrators and students.


1.2 Objective of the Project

The main objective of the Smart Parcel Management System is to develop a comprehensive web-based parcel tracking and management system that automates the entire parcel lifecycle from arrival to delivery at educational institutions. The project aims to create a centralized platform where administrators can efficiently register, track, and manage parcels while providing students with real-time visibility and secure collection mechanisms.

The major objectives of the project are:

- To provide secure role-based authentication system (Admin, Watchman, Student)
- To maintain comprehensive student profiles with contact and identification details
- To enable quick parcel registration with auto-fetch of student information
- To implement automated OTP generation for secure parcel handover verification
- To integrate email notification system for instant student alerts
- To provide real-time dashboard with parcel statistics and analytics
- To enable advanced search and filtering capabilities for parcel tracking
- To create a self-service student portal for parcel status checking
- To maintain complete audit trail of parcel movements and deliveries
- To eliminate manual record-keeping and reduce administrative workload
- To improve parcel security through OTP-based verification
- To enhance student satisfaction through instant notifications and transparency

The system helps bridge the gap between traditional manual processes and modern digital operations by providing an automated, secure, and user-friendly parcel management solution for educational institutions.


1.3 Need of the Project

In modern educational institutions, efficient administrative services are essential for student satisfaction and operational excellence. Parcel management, though often overlooked, is a critical service that affects every student living on campus or receiving deliveries. As e-commerce continues to grow, the volume of parcels arriving at educational institutions has increased dramatically, overwhelming traditional manual systems.

Many institutions struggle with:

- High volumes of parcels during festival seasons and sale periods
- Limited staff resources to manually process and track each package
- Student complaints about delayed notifications and lost packages
- Security concerns regarding package verification during collection
- Lack of data for optimizing mailroom operations and resource allocation

Educational institutions need a systematic digital platform to support modern parcel management requirements. Manual systems are difficult to scale, prone to errors, and cannot provide the real-time visibility that students expect in today's connected world.

Smart Parcel Management System is needed to provide a reliable and intelligent platform where staff can quickly register parcels, students receive instant email notifications with OTP verification codes, and administrators have complete visibility into operations through a comprehensive dashboard. It helps institutions improve service quality, reduce operational costs, enhance security, and deliver a superior experience to students.

The project also supports institutional goals of digital transformation by replacing paper-based processes with an efficient, eco-friendly digital solution that can generate valuable operational insights and analytics.

2. TECH STACK
================================================================================

The successful development of Smart Parcel Management System depends on the proper selection of technologies used for designing, developing, testing, and deploying the system. A technology stack refers to the combination of programming languages, frameworks, tools, servers, and database systems used to build a complete application.

Smart Parcel Management System is developed as a full-stack web application using frontend technologies for user interface design, backend technologies for server-side processing, and database technologies for secure data storage and management. Additional development tools and external services are used for coding, testing, debugging, email delivery, and deployment.

The selected technologies provide system reliability, scalability, security, and smooth communication between frontend and backend components.


2.1 Frontend Technologies

Frontend development focuses on creating the user interface and improving user interaction with the system. It is the part of the application that users directly see and use.

HTML5 (HyperText Markup Language)
HTML5 is used to create the semantic structure of web pages such as Role Selection Page, Admin Login Page, Student Login Page, Admin Dashboard, Manage Parcels Page, and Student Dashboard. It helps in organizing content like forms, buttons, tables, navigation menus, search panels, and user input fields. HTML5 semantic tags improve accessibility and SEO.

CSS3 (Cascading Style Sheets)
CSS3 is used for designing and styling the web pages with modern features like Flexbox and Grid layouts. It improves the visual appearance of the application by adding responsive layouts, colors, spacing, animations, shadows, and attractive user interface design. CSS variables and media queries ensure the application looks professional across all device sizes.

JavaScript (ES6+)
JavaScript is used for client-side scripting and dynamic content handling. It improves user interaction by validating forms, handling button actions, fetching data from backend APIs using Fetch API, updating content without page refresh, and implementing real-time search and filter functionality. ES6+ features like async/await, arrow functions, and template literals are used for cleaner code.


2.2 Backend Technologies

Backend development handles server-side processing, business logic, authentication, and communication with the database.

Node.js (v18+)
Node.js is used as the runtime environment for executing JavaScript on the server side. It provides non-blocking I/O and event-driven architecture that makes the application fast and scalable. Node.js handles multiple concurrent connections efficiently, making it ideal for real-time applications.

Express.js (v5.2.1)
Express.js is used as the web application framework for Node.js. It simplifies routing, middleware integration, request handling, and API development. Express.js provides a robust set of features for building RESTful APIs and handling HTTP requests and responses.

Key responsibilities:
- User Authentication and Session Management
- RESTful API Development
- Request Validation and Processing
- Database Communication
- OTP Generation Logic
- Email Integration
- Error Handling and Logging


2.3 Database Technology

The database stores all important project information such as student profiles, parcel records, OTP verification logs, and admin credentials.

MySQL (v8.x)
MySQL is used as the relational database management system for storing and managing all project data efficiently.

It stores:
- Student Details (name, SAP ID, phone, email, password hash)
- Admin Credentials (username, password hash, role)
- Parcel Records (parcel ID, student info, source, status, dates)
- OTP Records (parcel ID, OTP code, email, status, timestamps)

MySQL supports primary keys, foreign keys, normalization, constraints, and secure data management for reliable system performance. The database schema uses proper relationships between tables with CASCADE delete for data integrity.


2.4 External Services

Nodemailer (v8.0.5)
Nodemailer is used for sending automated email notifications to students when parcels arrive. It integrates with Gmail SMTP server to deliver OTP codes and parcel notifications securely.

Features:
- HTML email templates with professional styling
- Automatic OTP delivery to student email addresses
- Error handling for failed email deliveries
- Asynchronous email sending for improved performance


2.5 Development Tools

Various supporting tools are used during project development for coding, testing, debugging, and deployment.

Visual Studio Code
Visual Studio Code is used as the main code editor for writing HTML, CSS, JavaScript, and Node.js code. It provides useful features such as syntax highlighting, debugging support, extensions, and faster code management.

Google Chrome Developer Tools
Chrome Developer Tools are used for frontend testing, debugging, inspecting HTML/CSS elements, checking JavaScript errors, monitoring API responses, and analyzing network requests. It helps improve performance and identify development issues quickly.

Node Package Manager (npm)
npm is used for managing project dependencies including Express, MySQL2, bcrypt, cors, dotenv, and nodemailer. It simplifies installation and version management of required packages.

Git & GitHub
Git is used for version control to track code changes, manage branches, and collaborate effectively. GitHub hosts the remote repository for backup and sharing.


2.6 Server and Hosting Environment

Node.js Built-in Server
The application uses Node.js built-in HTTP server through Express.js to host the web application locally. It handles HTTP requests, serves static files, and executes API endpoints during development and testing.

Database Connectivity
The frontend communicates with the backend using JavaScript Fetch API, while the backend connects to MySQL using the mysql2 package with promise-based queries. The backend processes user requests and sends JSON responses back to the frontend for dynamic display.

This ensures smooth frontend-backend-database integration and real-time system performance.

3. METHODOLOGY

The methodology of Smart Parcel Management System explains the systematic approach followed for designing, developing, testing, and implementing the web-based parcel management system. A proper methodology ensures that the project is developed in an organized, efficient, and reliable manner.

Smart Parcel Management System is developed using a structured approach for secure user management, proper database design, smooth frontend-backend integration, and automated parcel tracking with OTP verification. The system provides parcel management based on student registration, parcel arrival logging, automatic email notifications, and secure handover verification.

The development process includes requirement analysis, system design, database creation, frontend development, backend integration, testing, and final deployment. This methodology helps in building a stable, scalable, and user-friendly application.


3.1 Methods Used

Smart Parcel Management System follows a modular and layered development approach to ensure better maintainability and performance. The main methods used in the project are:

- Requirement Analysis
- Three-Tier Architecture Design
- Database Normalization
- RESTful API Communication

3.2 Three-Tier Architecture

The project follows Three-Tier Architecture for better organization, scalability, security, and maintenance. This architecture divides the system into three main layers:

1. Presentation Layer
This is the frontend layer where users directly interact with the system. It provides a responsive and user-friendly interface for all project functionalities.

It includes:
- Role Selection Page
- Admin Login Page
- Student Login/Registration Page
- Admin Dashboard
- Manage Parcels Page
- Student Dashboard
- Handover Interface

Technologies Used:
- HTML5
- CSS3
- JavaScript (ES6+)

The presentation layer ensures smooth navigation, attractive design with red color theme (#d32f2f), and better user experience across desktop and mobile devices.

2. Application Logic Layer
This is the backend processing layer responsible for handling all system operations and business logic.

It manages:
- User Authentication (Student and Admin)
- Session Management (via localStorage)
- Parcel Registration and Tracking
- OTP Generation and Verification
- Email Notification Dispatch
- Database Operations
- Request Validation
- Search and Filter Processing

Technology Used:
- Node.js with Express.js

The logic layer processes user requests, applies business rules, communicates with the database, and sends email notifications asynchronously.

3. Database Layer
This layer stores and manages all project-related data securely and efficiently.

It includes:
- students table (user profiles)
- admins table (administrator accounts)
- parcels table (parcel records)
- parcel_otps table (OTP verification logs)

Technology Used:
- MySQL 8.x

The database layer ensures proper storage, retrieval, and management of information using relational database concepts with proper foreign key relationships.


3.3 RESTful API Design

The frontend communicates with the backend using JavaScript Fetch API and RESTful endpoints. This allows smooth data transfer and real-time interaction between the user interface and the server.

Examples of API endpoints include:

Authentication APIs:
- POST /api/auth/student/login - Student login with SAP ID
- POST /api/auth/admin/login - Admin login with credentials
- POST /api/auth/student/register - Student registration

Parcel APIs:
- GET /api/parcels - Get all parcels (admin)
- GET /api/parcels/student - Get parcels by student SAP ID/phone
- GET /api/parcels/search - Search parcels with filters
- POST /api/parcels - Add new parcel
- PUT /api/parcels/:id/deliver - Mark parcel as delivered

Student APIs:
- GET /api/students/search - Search student by name and phone

OTP APIs:
- POST /api/otp/generate - Generate OTP for parcel
- POST /api/otp/verify - Verify OTP and mark delivered

Dashboard APIs:
- GET /api/dashboard/stats - Get parcel statistics

These endpoints process requests, retrieve required data from the database, and return responses in JSON format for frontend display. This improves system efficiency and supports dynamic content updates without unnecessary page reloads.


3.4 Database Design Techniques

The database is designed using normalization techniques to reduce redundancy and improve data consistency. Proper relational structure is maintained using primary keys and foreign keys.

Main Tables Used:
- students (id, name, sap_id, phone, email, password_hash, timestamps)
- admins (id, username, password_hash, role, timestamps)
- parcels (id, parcel_id, student_id, student_name, sap_id, phone, email, source, delivery_person, delivery_phone, status, arrival_date, delivery_date, timestamps)
- parcel_otps (id, parcel_id, email, otp_code, status, created_at, expires_at, used_at)

Database Features Include:
- Primary Keys for unique identification
- Foreign Keys for relationship management
- CASCADE DELETE for referential integrity
- ENUM constraints for status fields
- NOT NULL constraints for required fields
- UNIQUE constraints for SAP ID and phone
- DATETIME fields for audit trails
- Automatic timestamp updates

These techniques ensure secure, accurate, and organized data handling throughout the system.


3.5 Security Implementation

The system implements multiple security measures:

Password Security:
- bcrypt hashing with salt rounds (10) for password storage
- No plaintext password storage in database

Input Validation:
- Server-side validation for all inputs
- Regex patterns for SAP ID (11 digits), phone (10 digits), email format
- SQL injection prevention through parameterized queries

Authentication:
- Role-based access control (Student, Admin, Watchman)
- Client-side session management using localStorage
- Protected routes with login checks

OTP Security:
- 6-digit random OTP generation
- OTP invalidation after use
- Parcel-specific OTP binding

4.1 Step-by-Step Working

1. User Access and Role Selection
The user opens the application and is presented with a Role Selection page offering three options: Admin, Watchman/Staff, and Student.

For Admin/Watchman:
- Navigate to Admin Login page
- Enter username and password (default: nmims@123 / nmims123)
- System verifies credentials using bcrypt hashing
- On successful login, redirected to Admin Dashboard

For Student:
- New students navigate to Registration page
- Enter name, SAP ID (11 digits), phone (10 digits), email, and password
- System validates all inputs and checks for duplicates
- On successful registration, redirected to Student Login
- Existing students login with SAP ID and password
- On successful login, redirected to Student Dashboard

Screenshot: Role Selection Page, Admin Login Page, Student Registration Page


2. Admin Dashboard Navigation
After successful login, the administrator enters the Dashboard where comprehensive parcel management capabilities are available.

The Dashboard displays:
- Summary statistics (Total Parcels, Delivered, Pending)
- Quick add parcel form with auto-fetch student details
- Navigation to Manage Parcels page
- Clear All Parcel Data functionality

The admin can:
- View real-time statistics
- Add new parcels with student lookup
- Navigate to detailed parcel management
- Monitor operational metrics

Screenshot: Admin Dashboard


3. Parcel Registration Process
When a parcel arrives, the admin uses the "Add New Parcel" form:

Step 1: Enter student name and phone number
Step 2: Click "Fetch Details" button
Step 3: System searches database and auto-fills:
       - SAP ID
       - Student Email
Step 4: Enter parcel source (Amazon, Flipkart, etc.)
Step 5: Enter delivery person name and phone
Step 6: Submit form

The system:
- Validates all required fields
- Generates sequential parcel ID (1, 2, 3...)
- Saves parcel record with "Arrived" status
- Updates dashboard statistics
- Shows success notification

Screenshot: Add Parcel Form with Fetch Details


4. Manage Parcels and OTP Generation
The Manage Parcels page provides comprehensive parcel tracking and OTP-based handover functionality.

Features available:
- Search and filter by date, parcel ID, student details
- View all parcels in tabular format
- Generate OTP for parcel handover
- Verify OTP and mark as delivered
- View delivery history

OTP Generation Process:
- Admin clicks "Generate OTP" for a parcel
- System generates 6-digit random OTP
- Saves OTP to database with "Active" status
- Sends OTP to student's registered email
- Displays OTP on screen for admin reference
- OTP remains valid until parcel is collected

Screenshot: Manage Parcels Page with OTP Section


5. OTP Verification and Delivery
When student arrives to collect parcel:

Step 1: Student provides OTP to admin
Step 2: Admin enters OTP in verification field
Step 3: Admin clicks "Verify" button
Step 4: System validates OTP against database
Step 5: On successful verification:
       - Marks OTP as "Used"
       - Updates parcel status to "Delivered"
       - Records delivery timestamp
       - Updates statistics

Screenshot: OTP Verification Success


6. Student Portal Experience
Students access their personal dashboard to view parcel status:

After login, Student Dashboard shows:
- Welcome message with student name
- Summary cards (Total, Arrived, Delivered)
- Complete parcel history table
- Parcel details including arrival date, source, status
- Delivery date for collected parcels

Students can:
- View all their parcels in one place
- Check current status without visiting mailroom
- See delivery confirmation
- Plan collection based on arrival notifications

Screenshot: Student Dashboard


4.2 Project Execution Steps

The development of Smart Parcel Management System was completed through a systematic execution process to ensure proper planning, implementation, testing, and deployment.

1. Requirement Analysis
The project started by identifying the need for a digital parcel management system at educational institutions. The main objective was to solve manual tracking inefficiencies and provide automated notifications with secure verification.

2. System Design
The system was designed using Three-Tier Architecture with separate frontend, backend, and database layers for better scalability, security, and maintenance. This design improves performance and makes future updates easier.

3. Database Design
The MySQL database was created using normalized tables:
- students (user profiles with authentication)
- admins (administrator accounts)
- parcels (parcel records with status tracking)
- parcel_otps (OTP verification logs)

Primary keys and foreign keys were used to maintain relationships and ensure data consistency. CASCADE delete ensures referential integrity when records are removed.

Screenshot: Database Schema in MySQL

4. Frontend Development
Frontend pages were developed using:
- HTML5 for semantic structure
- CSS3 with Flexbox/Grid for responsive design
- JavaScript ES6+ for dynamic functionality

Pages created:
- Role Selection (index/landing)
- Admin Login
- Student Login and Registration
- Admin Dashboard
- Manage Parcels
- Student Dashboard

The design follows a consistent red color theme (#d32f2f) for branding and visual identity.

5. Backend Development
Node.js with Express.js was used for backend development to handle:
- User Authentication (bcrypt password hashing)
- RESTful API Development
- Database Operations (MySQL2 with promises)
- OTP Generation Logic
- Email Integration (Nodemailer with Gmail SMTP)
- Input Validation and Error Handling

The backend ensures smooth server-side processing and proper functionality of the application.

6. Integration
Frontend and backend were connected using JavaScript Fetch API and JSON responses for smooth and real-time data transfer. This allows users to interact with the system without unnecessary page reloads and improves user experience.

7. Testing
The system was tested using:
- Unit Testing (individual API endpoints)
- Integration Testing (frontend-backend communication)
- Database Testing (query performance and constraints)
- UI Testing (responsive design across devices)
- Email Testing (OTP delivery verification)

Testing ensured proper functionality, security, reliability, and performance of the final application.

8. Final Implementation
The complete project was deployed using Node.js server with MySQL database support. The application runs on localhost:3000 by default and provides a fully functional web-based parcel management system ready for practical use in educational institutions, hostels, and residential complexes.

Smart Parcel Management System is a comprehensive web-based parcel tracking and management solution developed to modernize parcel handling operations at educational institutions. Many institutions continue to rely on manual paper-based systems that are inefficient, error-prone, and unable to provide the real-time visibility that students expect. This project solves these problems by providing a digital platform that automates parcel registration, provides instant email notifications with OTP verification, and offers comprehensive administrative controls.

Smart Parcel Management System successfully combines Web Programming and Database Management System concepts into a complete full-stack application.

The project can be effectively used by colleges, universities, hostels, and residential complexes to modernize their parcel management operations. It provides a strong foundation for digital transformation of administrative services and helps institutions improve operational efficiency, reduce costs, enhance security, and deliver superior service quality to students and residents.
